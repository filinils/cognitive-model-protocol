# Contributing

CMP is a draft protocol seeking criticism.

Contributions are welcome in the form of issues, pull requests, examples, schema improvements, ethical concerns, implementation notes, and alternative formulations.

Please prioritize critique before expansion. The project should become clearer before it becomes larger.

## Contribution Guidelines

- Challenge assumptions before expanding scope.
- Do not add personality labels unless they are represented as hypotheses with evidence and uncertainty.
- Prefer falsifiable model elements.
- Preserve the distinction between a `collaboration_model` and a `user_profile`.
- Make uncertainty visible.
- Identify ethical risks early.
- Prefer practical interoperability over theoretical completeness.

## Useful Contributions

- Concrete examples of collaboration models.
- Cases where the schema overreaches.
- Cases where the schema is too weak to express needed uncertainty.
- Suggestions for user challenge, correction, deletion, and export flows.
- Comparisons with related formats or memory systems.
- Ethical objections and misuse scenarios.

## Pull Requests

Pull requests should explain:

- What problem the change addresses.
- Which assumptions it changes.
- Whether it affects the schema, examples, or terminology.
- What alternatives were considered.

Large changes may later move through a CMP RFC process. Until then, issues and pull requests are sufficient.
